从持仓输入到证据化研究工作台

产品输入/输出 API/UI Agent 工作流 数据/监控

用户入口

Position 输入 公司名、ticker、市场、持有或关注、数量和成本。
解析确认 LLM 解析规范实体，用户确认后才落库，失败可降级按原样添加。
组合首页 持仓列表、运行状态、组合洞察、共享供应链、相关性矩阵。

LangGraph 研究闭环

1 intake_resolve解析持仓为图谱 seed entity。
2 ingest搜索和来源抓取，Tavily 或降级数据。
3 filter过滤与目标无关的候选证据。
4 evidence_build写 source documents、evidences、FTS 和 Chroma。
5 expand产生产业链边，标 basis、confidence、evidence。
6 intel_synth生成中文情报条目，保留证据 ID。
7 thesis_draft生成 thesis summary 和 assumption。
8 risk_review拦无证据 claim、荐股、目标价和预测。
9 human_confirm等待用户确认或编辑 thesis。
10 finalize落库并结束 run，前端继续轮询状态。
Expand Run 节点一跳懒加载，复用 intake、ingest、filter、evidence、expand、intel、finalize，跳过 thesis 和确认。
Grounding Contract 用户可见结论必须带 evidence_ids，source_backed 不能为空证据，inferred 必须显示 confidence。

Maintenance

后续维护检查点

  • 1新增 LangGraph 节点时，同步更新中间工作流、trace 监控字段和验收说明。
  • 2新增 API 路由或 UI 入口时，同步更新左侧用户入口、右侧交付物和核心功能覆盖。
  • 3新增数据库表时，确认它属于业务事实、AgentOps、cache 还是 eval，并补到数据契约说明。
  • 4调整模型或价格时，同步更新成本默认值，并确认 /metrics/summary 仍可回算旧 run。
  • 5做阶段收口时，先更新项目 devlog 和 index，再更新 Brain 高层日志。
