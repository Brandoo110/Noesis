# Noesis — Claude Code 工作指南

> 30 秒认知：**Noesis** 是一个"投资情报 Agent"。输入你的持仓（任意市场）→ AI 调研公司 + 产业链 + 情报/舆情/方向（每条带证据）→ 自动派生 thesis 给你确认 → 沿产业链图谱自由下钻。**调研工具，非荐股工具。**
> 技术栈：Python / LangGraph / FastAPI / SQLite。新业务代码进 `graph/`（研究流节点）、`db/`（领域表）、`api/`、`web/`（图谱前端）。

---

## 协作角色（本项目核心约定）

- **Claude Code = 主控 / 设计 / 验收**：拆任务、写/审 spec、验收 Codex 产出，**不直接当代码作者**。
- **Codex = 代码作者**：按 `AGENTS.md`（完整编码契约）落地实现。
- 分工不重复：编码细节（命名、结构、测试规范）一律在 `AGENTS.md`；本文件只管"做什么 / 不做什么 / 验收什么 / 去哪找"。

## Context Tiers（指针，不是图书馆）

- **Tier 1（每次加载）**：本 `CLAUDE.md` —— 项目是什么、红线、角色、验收口径。
- **Tier 2（按需读取）**：
  - `docs/specs/2026-06-24-noesis-v1-design.md` —— **V1 冻结设计**（架构/领域模型/研究流/取舍）。
  - `AGENTS.md` —— 编码契约（Codex 落地依据）。
- **Tier 3（参考/默认不碰）**：vault `Projects/active/Noesis/README.md` + `EVALUATION.md` —— 早期另一套构思，**不代表当前定位**，除非明确要求不引用。

## Do NOT（与"要做什么"同等重要）

- **不荐股**：不输出买/卖/加减仓建议、目标价、涨跌预测；不接任何交易/资金通道。投资相关只能以"研究待办/关注点（仅供参考）"小字形式出现。【产品红线，违反即不验收】
- **不做 GraphRAG / 图数据库**（Neo4j 等）：V1 用 SQLite 邻接表 + 证据引用式检索（本地向量+FTS）。理由见 spec §10。
- **不做"全图预研"**：产业链一律**懒加载**，节点点到才研究，结果缓存。禁止一次性研究整张产业图。
- **不允许无证据断言**：每条结论必须挂 evidence；LLM 推断的产业链边必须标 `inferred`，与一手证据视觉区分。
- **不移植 PulseRadar 的 brand / competitor / 品牌增长业务域**：只移植通用 backbone（编排骨架/SQLite thin repo/trace/grounding/risk_reviewer/eval/确认闸门/报告模板）。
- **不重型化**：不引入 Kafka / Flink / ClickHouse / Elasticsearch / K8s。本地优先、单人可建。

## 主控验收 checklist（接收 Codex 产出前必查）

读完一段代码/PR，5 秒内能判断是否符合：

1. 红线无违反（无荐股/预测/交易语义）。
2. 每条结论可溯源；产业链 `inferred` 标注诚实、`source_backed` 真有出处。
3. 懒加载生效（无全图预研、有缓存命中）。
4. 研究流 pytest 绿（happy-path + 降级路径）。
5. 未重复 `AGENTS.md` 已定的编码细节；新增依赖未触犯 Do NOT。

## 工作风格（用户偏好）

- 中文回复，**禁止韩文或其他非中文/英文内容**。
- 先给方案，不直接写代码；不确定时列选项不猜测。重大变更先问，小优化可直接做。
- 简洁，不冗余。本地优先、轻量、不过度工程化。
- 文件路径用绝对路径。不留注释掉的代码块 / 调试 print。
